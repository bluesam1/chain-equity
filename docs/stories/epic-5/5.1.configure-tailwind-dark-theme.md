# Story 5.1: Configure Tailwind CSS with Dark Theme

## Status
Draft

## Story

**As a** frontend developer,
**I want** Tailwind CSS configured with the dark theme color palette from the Front-End Specification,
**so that** I can build UI components that match the design system requirements.

## Acceptance Criteria

1. Tailwind CSS configured with dark theme colors from Front-End Specification (`docs/front-end-spec.md`)
2. Color palette implemented per specification:
   - Background: `#0F172A` (Slate-900)
   - Surface: `#1E293B` (Slate-800)
   - Border: `#334155` (Slate-700)
   - Text Primary: `#F1F5F9` (Slate-100)
   - Text Secondary: `#CBD5E1` (Slate-300)
   - Text Muted: `#94A3B8` (Slate-400)
   - Primary: `#3B82F6` (Blue-500)
   - Success: `#10B981` (Green-500)
   - Warning: `#F59E0B` (Amber-500)
   - Error: `#EF4444` (Red-500)
3. Typography scale configured per specification (H1-H3, Body, Small, Caption)
4. Spacing scale configured per specification (base unit: 0.25rem/4px)
5. Container max-width set to 1280px (xl breakpoint)
6. Dark theme applied as default theme
7. All colors accessible via Tailwind utility classes

## Tasks / Subtasks

- [ ] Configure Tailwind CSS color palette (AC: 1, 2)
  - [ ] Add custom colors to Tailwind config or CSS variables
  - [ ] Map all colors from Front-End Specification
  - [ ] Ensure color names match specification (background, surface, border, text-primary, etc.)
  - [ ] Verify all colors are accessible via Tailwind classes
- [ ] Configure typography scale (AC: 3)
  - [ ] Set font families (Inter or System UI for primary, JetBrains Mono/Fira Code for monospace)
  - [ ] Configure type scale per specification (H1: 2.25rem/36px, H2: 1.875rem/30px, etc.)
  - [ ] Configure font weights per specification
  - [ ] Configure line heights per specification
- [ ] Configure spacing scale (AC: 4)
  - [ ] Set base unit to 0.25rem (4px)
  - [ ] Configure common spacing values (0.5rem, 1rem, 1.5rem, 2rem)
  - [ ] Configure section spacing (2rem between major sections)
  - [ ] Configure card padding (1.5rem internal padding)
- [ ] Configure layout settings (AC: 5)
  - [ ] Set container max-width to 1280px (xl breakpoint)
  - [ ] Configure consistent padding (1rem base, 1.5rem for cards)
- [ ] Apply dark theme as default (AC: 6)
  - [ ] Set dark background color as default
  - [ ] Configure text colors for dark theme
  - [ ] Verify dark theme is active by default
- [ ] Verify Tailwind configuration (AC: 7)
  - [ ] Test all color classes work correctly
  - [ ] Verify typography classes work correctly
  - [ ] Verify spacing classes work correctly
  - [ ] Build and verify no errors

## Dev Notes

### Relevant Source Tree Info
- Frontend workspace: `frontend/`
- Tailwind CSS config: `frontend/tailwind.config.js` (if using v3) or CSS variables in `frontend/src/index.css` (v4)
- Main CSS file: `frontend/src/index.css`
- Tailwind CSS v4 uses `@tailwindcss/vite` plugin (no config file needed, uses CSS)

### Front-End Specification References
- **Color Palette:** See `docs/front-end-spec.md` - Branding & Style Guide > Color Palette section
- **Typography:** See `docs/front-end-spec.md` - Branding & Style Guide > Typography section
- **Spacing & Layout:** See `docs/front-end-spec.md` - Branding & Style Guide > Spacing & Layout section
- **Design System:** See `docs/front-end-spec.md` - Component Library / Design System section

### Architecture Decisions
- Tailwind CSS v4 with `@tailwindcss/vite` plugin (no config file needed for v4)
- Colors defined as CSS custom properties or Tailwind theme extension
- Dark theme is default (no light theme support for now)
- Typography scale follows Front-End Specification exactly
- Spacing scale follows Front-End Specification exactly

### Key Constraints
- Must use exact color values from Front-End Specification
- Must use exact typography scale from Front-End Specification
- Must use exact spacing scale from Front-End Specification
- Dark theme only (no light theme)
- All colors must be accessible via Tailwind utility classes

### Testing
- **Test File Location**: Manual verification in browser
- **Test Standards**: Visual verification against Front-End Specification
- **Testing Frameworks**: N/A (configuration story)
- **Specific Requirements**: 
  - Verify all colors match specification
  - Verify typography scale matches specification
  - Verify spacing scale matches specification
  - Verify dark theme is applied correctly
  - Test Tailwind classes work in components
