# Story 1.3: Set Up Frontend Workspace

## Status
Ready for Review

## Story

**As a** frontend developer,
**I want** a Vite.js + React + TypeScript frontend workspace configured,
**so that** I can build a modern web interface with MetaMask integration and Supabase authentication.

## Acceptance Criteria

1. Vite.js project initialized in `frontend/` workspace with React + TypeScript template
2. Required dependencies installed: React, Vite, TypeScript, ethers.js, Supabase client, Tailwind CSS
3. TypeScript configuration (`tsconfig.json`) configured for React
4. Tailwind CSS configured and set up
5. Vite configuration (`vite.config.ts`) set up
6. Project structure created: `src/pages/`, `src/components/`, `src/lib/`
7. Development server can be started and runs successfully

## Dev Notes

### Relevant Source Tree Info
- Frontend workspace: `frontend/`
- Entry point: `frontend/src/main.tsx`
- Pages: `frontend/src/pages/`
- Components: `frontend/src/components/`
- Utilities: `frontend/src/lib/`
- Hooks: `frontend/src/hooks/`
- Types: `frontend/src/types/`
- Vite config: `frontend/vite.config.ts`
- TypeScript config: `frontend/tsconfig.json`
- Tailwind CSS v4: Uses @tailwindcss/vite plugin (no config file needed)

### Architecture Decisions
- Vite.js for fast development and optimized builds
- React 18.x for UI framework
- TypeScript for type safety
- Tailwind CSS v4 (latest version) for styling - high-performance engine, modern CSS features
- ethers.js v6 for blockchain interactions
- Supabase Web3 Auth for authentication
- wagmi for wallet connection

### Key Constraints
- Must use Vite.js (not Create React App or Next.js)
- Must use TypeScript throughout
- Must use Tailwind CSS (latest version)
- Environment variables must use `VITE_` prefix (Vite requirement)
- Must use ethers.js v6 (not v5)

### Testing
- **Test File Location**: `frontend/src/` (co-located with components) or `frontend/__tests__/`
- **Test Standards**: Use Vitest or React Testing Library
- **Testing Frameworks**: Vitest (Vite's test runner) or Jest with React Testing Library
- **Specific Requirements**: 
  - Tests should be written in TypeScript
  - Component tests should use React Testing Library
  - Test files should follow naming convention: `*.test.tsx` or `*.spec.tsx`
